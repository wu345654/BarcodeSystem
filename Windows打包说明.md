# Windows版本打包说明

## 准备工作

1. **安装Python 3.11**
   - 从官方网站下载并安装 Python 3.11：https://www.python.org/downloads/windows/
   - 安装时勾选"Add Python to PATH"

2. **下载项目文件**
   - 将 `BarcodeSystem.zip` 解压到任意目录

3. **安装依赖**
   - 打开命令提示符（CMD）
   - 进入项目目录
   - 运行：`pip install flask flask-cors python-barcode Pillow==9.5.0 pyinstaller`

## 打包步骤

1. **运行打包脚本**
   - 在项目目录中双击运行 `build_windows.py`
   - 或在命令提示符中运行：`python build_windows.py`

2. **等待打包完成**
   - 脚本会自动清理旧文件、安装依赖、打包可执行文件
   - 打包完成后，可执行文件会生成在 `dist` 目录中

## 运行系统

### 方法一：使用打包后的可执行文件
1. **双击运行**
   - 进入 `dist` 目录
   - 双击 `BarcodeSystem.exe` 运行系统

2. **访问系统**
   - 系统会自动在浏览器中打开 `http://127.0.0.1:888`
   - 或手动在浏览器中输入该地址

### 方法二：一键运行脚本
1. **直接运行**
   - 在项目根目录中双击 `run_windows.bat`
   - 脚本会自动检查依赖、安装必要的包并启动系统

2. **便捷性**
   - 无需手动安装依赖
   - 一键启动，适合快速使用
   - 提供详细的启动信息和功能说明

## 系统功能

- **订单管理**：创建、编辑、删除订单
- **条码生成**：根据订单数量自动生成唯一条码
- **条码扫描**：支持扫描枪和手动输入，防止重复扫描
- **扫描记录**：查看所有扫描历史
- **统计报表**：基于扫描结果生成订单统计信息
- **条码打印**：支持80mm*50mm纸张大小的标签打印

## 常见问题

1. **端口被占用**
   - 错误信息：`Address already in use`
   - 解决方案：关闭占用888端口的其他程序，或修改 `start.py` 中的端口号

2. **缺少依赖**
   - 错误信息：`ModuleNotFoundError`
   - 解决方案：重新运行 `pip install` 命令安装缺失的依赖

3. **数据库文件**
   - 系统会自动创建 `order_system.db` 数据库文件
   - 如需备份数据，复制此文件即可
